Building FindBugs from source

This document describes how to build FindBugs from source.

Acknowledgments

The Ant script used to build FindBugs was generously contributed by Mike Fagan.

Prerequisites

You will need the following:

  • The FindBugs source distribution
  • The latest CVS version of BCEL with the latest FindBugs patch applied; this is included in the FindBugs binary distribution (see note below)
  • JDK 1.4.0 or later
  • The GJ Compiler; this is an early-access version of the generics-enabled javac.
  • Apache Ant, version 1.5 or later
  • The JSR 14 adapter for Ant, available from this web page.  Put the file jsr14adapter-1.1.jar in the lib subdirectory of your Ant installation.

BCEL

You must use the version of bcel.jar that is included with the binary distribution of FindBugs.  That version contains numerous bug fixes and design improvements that are not available in the official BCEL sources.  If you really want to compile BCEL yourself, then you need to download this patch and apply it before building FindBugs.

Steps

  1. Unpack the source distribution; this will create directories findbugs and findbugsGUI.
  2. Copy bcel.jar from the FindBugs binary distribution into the findbugsGUI/lib directory.
  3. Change your PATH so the javac script from the GJ early-access release is your default javac.
  4. Change directory to the findbugsGUI directory.
  5. Invoke the ant command.  You will see some messages scroll by.  Eventually, you should see the output BUILD SUCCESSFUL.

If all of the above steps succeed, you should be able to run the GUI and text UI versions of FindBugs using the commands 

  java -jar build/lib/findbugsGUI.jar
  java -jar build/lib/findbugs.jar
