Building FindBugs from source

This document describes how to build FindBugs from source.

Prerequisites

You will need the following:
  • The FindBugs source distribution (if you are reading this document, you probably already have it)
  • The latest CVS version of BCEL with the latest FindBugs patch applied; see below
  • GNU Make
  • JDK 1.4.0 or later
  • The GJ Compiler; see the link labeled "Adding Generics to the Java(TM) Programming Language v. 1.3"

BCEL

You must use the version of bcel.jar that is included with the binary distribution of FindBugs.  That version contains numerous bug fixes and design improvements that are not available in the official BCEL sources.  If you really want to compile BCEL yourself, then you need to download this patch and apply it before building FindBugs.

Steps

  1. Add bcel.jar to your CLASSPATH environment variable
  2. Change your PATH so the javac script from the GJ early-access release is your default javac
  3. Go to the findbugsGUI subdirectory of the FindBugs source distribution
  4. Run the command make (GNU Make is required here)
  5. Copy bcel.jar into the lib subdirectory of findbugsGUI

If all of the above steps succeed, you should be able to run the GUI and text UI versions of FindBugs using the commands 

  java -jar lib/findbugsGUI.jar
  java -jar lib/findbugs.jar
