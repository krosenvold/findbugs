FindBugs FAQ

This document contains answers to frequently asked questions about FindBugs, a bug pattern detector for Java.

Q1: I'm getting java.lang.UnsupportedClassVersionError when I try to run FindBugs

FindBugs requires JRE 1.4.0 or later to run.  If you use an earlier version, you will see an exception error message similar to the following: 

Exception in thread "main" java.lang.UnsupportedClassVersionError:
edu/umd/cs/findbugs/gui/FindBugsFrame (Unsupported major.minor version 48.0)
The solution is to upgrade to JRE 1.4.0 or later.

Q2: When I click the "Find Bugs!" button, I get a NoSuchMethodError

The symptom of this bug is that when you start the FindBugs analysis, you see an exception similar to the following: 

java.lang.NoSuchMethodError: org.apache.bcel.Repository.setRepository(Lorg/apache/bcel/util/Repository;)V
        at edu.umd.cs.findbugs.FindBugs.clearRepository(FindBugs.java:483)
        ...
The problem here is that the wrong version of the Apache BCEL library is being found.  The probable cause is that you have installed libraries from the Java(TM) Web Services Developer Pack 1.2 in the lib/endorsed directory of your Java Runtime Environment (JRE).  The file xsltc.jar contains an old version of BCEL that is incompatible with FindBugs.

Unfortunately, there is no easy solution to this problem, because files in the lib/endorsed directory take precedence over those those in the application's CLASSPATH.  Therefore, FindBugs will not worked with these Jar files installed in this way.  A workaround is to move these Jar files into a different directory, and use the java.endorsed.dirs system property to specify the new location for programs that require these files.

Many thanks to Peter Meulmeester for tracking down the cause of this problem.
