FindBugs - A Bug Pattern Detector for Java

This is the web page for FindBugs, a program which looks for bugs in Java code.  It is free software, distributed under the terms of the Lesser GNU Public License.

FindBugs requires JRE (or JDK) 1.4.0 or later to run.  However, it can analyze programs compiled for any version of Java.

The current version of FindBugs is @VERSION@, released on @RELEASE_DATE@.

FindBugs is beta quality software.  You may find bugs in it, and the features and source code may change significantly in future releases.  We are very interested in getting feedback on how to improve FindBugs.  Send your comments to daveho@cs.umd.edu.

News

David Hovemeyer and Bill Pugh will present an updated version of Finding Bugs is Easy as part of the Onward! track of OOPSLA 2004.  A draft of the paper is available.

Nick Rutar, Christian Almazan, and Jeff Foster of the University of Maryland Computer Science Department will present a paper, A Comparison of Bug Finding Tools for Java, at ISSRE 2004.  The paper compares several Java bug checkers, including FindBugs.

Changes since version 0.8.5:

  • New detector to find calls to Thread.interrupted() in a non static context, and especially with non currentThread() threads. (Dave Brosius)
  • New detector to find calls to .equals methods that use Object's version. (Dave Brosius)
  • Extend Dumb Method detector for toUpperCase, toLowerCase without a locale, and new Integer(1).toString() (Dave Brosius)
  • New detector to find Applets that call methods in the constructor refering to the AppletStub (Dave Brosius)
  • Detector to find infinite recursion (Bill Pugh)
  • New Style category for detectors (David Hovemeyer)
  • Ability to annotate methods using jdk5.0 annotations that suppress FindBugs warnings (Bill Pugh)
  • New -adjustExperimental for lowering priority of BugPatterns that are experimental (Dave Brosius)
  • Allow for command line options 'files' using the @ symbol (David Hovemeyer)
  • New -adjustPriority cmd line option to for adjusting bug priorites (David Hovemeyer)
  • Opcode parameter stack class for tracking types and values of stack items (Dave Brosius)
  • Added an Edit menu (cut/copy/paste) to gui (Dave Brosius)
  • French translation supplied (David Cotton)
