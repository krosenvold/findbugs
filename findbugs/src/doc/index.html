FindBugs™ - Find Bugs in Java Programs

This is the web page for FindBugs, a program which looks for bugs in Java code.  It is free software, distributed under the terms of the Lesser GNU Public License. The name FindBugs™ and the Findbugs logo are trademarked by The University of Maryland.

FindBugs is sponsored by Fortify Software, which sells security tools, including Fortify Source Code Analysis, which scans for security vunerabilities (press release).

FindBugs requires JRE (or JDK) 1.4.0 or later to run.  However, it can analyze programs compiled for any version of Java. The current version of FindBugs is @VERSION@, released on @RELEASE_DATE@.

We are very interested in getting feedback on how to improve FindBugs.  Send your comments to bassclar@users.sourceforge.net.

News

Changes since verson 1.0.0:

  • This is a preliminary release. Subsequent releases will more extensively document changes in this space. For now
    • FindNullDerefs reports null values that are guaranteed to be dereferenced (except on -effort:min)
    • Prelimary support for detectors using the ASM bytecode framework. You may experiment with writing ASM-based detectors, but beware the API may still change (which could possibly also affect BCEL-based detectors).
    • Bug categories (CORRECTNESS, MT_CORRECTNESS, etc.) are no longer hard-coded. Several bug patterns have been moved from CORRECTNESS and STYLE into a new category, BAD_PRACTICE.
