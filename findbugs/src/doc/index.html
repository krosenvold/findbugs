FindBugs - A Bug Pattern Detector for Java

This is the web page for FindBugs, a program which looks for bugs in Java code.  It is free software, distributed under the terms of the Lesser GNU Public License.

FindBugs requires JRE (or JDK) 1.4.0 or later to run.  However, it can analyze programs compiled for any version of Java.

Please note that FindBugs is beta-quality software.  You may find bugs in it, and the features and source code may change significantly in future releases.  Having said that, we invite you to try it out, and especially send us feedback.

Changes since last version

Changes since version 0.7.2:

  • Documentation includes an HTML document with descriptions of all bug patterns reported by FindBugs
  • Changed the Inconsistent Synchronization detector so that fields synchronized 50% of the time (or more) are reported as medium priority bugs (previously they were reported as low)
  • New detector to find private methods that are never called
  • New detector to find suspicious uses of non-short-circuiting boolean operators (& and |, rather than && and ||)
  • FindBugs web page has a link to a Japanese translation of the FindBugs manual, contributed by Hiroshi Okugawa

Contributing

If you have a bug fix or feature enhancement you would like to contribute, we would be happy to consider it for inclusion.  Just send a patch (be sure to use the "-u" or "-c" option to diff) or send the source files you've modified to daveho@cs.umd.edu.

Note: if you are contributing a new bug detector or an enhancement to an existing one, please submit a test case that contains an example of the new kind of bug.

Resources

There are two mailing lists for FindBugs.

  • Findbugs-announce is a low volume (moderated) list for announcements of new releases.
  • Findbugs-discuss is for discussion of planned features, bugs, development issues, etc.  Note that you must be a subscriber in order to post messages to the list.

Send comments to daveho@cs.umd.edu
